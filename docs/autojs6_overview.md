# AutoJs6 概览说明

> 本文是对 AutoJs6 官方文档 **Overview - 综述** 的简要整理，仅作为本项目内的快速说明。
> 详细、最新内容请以官网为准：<https://docs.autojs6.com/#/overview>

---

## 1. AutoJs6 是什么？

- **Android 平台上的自动化脚本引擎与开发环境**。
- 使用 **JavaScript / TypeScript 语法** 编写脚本。
- 通过 **无障碍服务 / Shizuku / ADB 等方式** 操控手机界面：点击、滑动、输入文字、读取界面控件信息等。
- 适合做：日常操作自动化、应用自动化测试、批量任务脚本、简单的桌面工具等。

与传统 UIAutomator / Appium 相比，AutoJs6 更偏向“脚本工具 + 个人自动化”，上手成本低、迭代速度快。

---

## 2. 主要特性概览

- **JavaScript/TypeScript 运行环境**  
  内置 JS 引擎，支持大部分 ES 语法特性，可使用模块、函数、类等组织代码。

- **UI 自动化能力**  
  - 查找界面控件（文本、ID、描述、坐标等多种选择器）。  
  - 模拟点击、长按、滑动、输入文本等操作。  
  - 读取控件属性，用于逻辑判断和测试断言。

- **权限与执行方式多样**  
  - 使用无障碍服务进行自动化（常见模式）。  
  - 支持 Shizuku / ADB 等更高权限方式，能进行更底层的操作（以官方文档为准）。

- **脚本工程化支持**  
  - 模块化脚本组织。  
  - 日志输出、异常捕获、调试功能。  
  - 结合第三方库，可扩展网络请求、文件操作等能力。

## 3. 基本使用流程

以下为典型的 AutoJs6 使用流程，仅为示意，具体请查看官方文档和示例脚本。

1. **在手机上安装 AutoJs6 客户端**  
   从可信渠道下载安装，并授予必要权限（无障碍、悬浮窗等）。

2. **编写脚本**  
   - 在 AutoJs6 内置编辑器中直接编写 JS/TS 脚本；  
   - 或使用外部编辑器（如 VS Code）编写，然后同步到手机。

3. **运行与调试**  
   - 在客户端中选择脚本运行；  
   - 观察日志输出、Toast 提示或 UI 变化；  
   - 根据需要调整等待时间、选择器条件等。

4. **打包与分发（可选）**  
   某些场景可以将脚本或工程打包为 APK，供他人安装使用（以官方说明为准）。

## 4. 典型应用场景

- **应用功能自动化测试**：
  - 批量登录、跳转页面、填写表单、检查界面元素是否存在。
- **日常操作自动化**：
  - 自动打卡、自动签到、自动抢券、定时执行某些任务等。
- **批量处理任务**：
  - 批量处理图片、批量发送消息、批量导出/录入数据等。
- **个人小工具**：
  - 自制简单 UI 工具，封装常用自动化操作给自己使用。

## 5. 与本项目（Python UIAutomatorViewer）的结合建议

本项目用于 **分析 Android 界面层级（UIAutomator XML + 截图）**，帮助你更准确地编写 AutoJs6 脚本选择器。

推荐的一种工作流：

1. **用 ADB 抓取当前界面 XML 与截图**（通过本工具的“刷新”获取）。
2. **在右侧控件树中选择目标控件**，查看：
   - `text`：控件显示文本；
   - `id` / `fullid`：资源 ID，如 `com.xx.app:id/btn_ok`；
   - `class`：控件类名，如 `android.widget.TextView`；
   - `bounds` / `center`：控件矩形与中心坐标等。
3. **根据控件属性，在 AutoJs6 中编写选择器**，例如：
   - `text("确定").click()`  
   - `id("btn_ok").click()`  
   - `className("android.widget.TextView").text("标题")`  等。
4. 在 AutoJs6 中运行脚本，若定位不稳定，可回到本工具继续分析界面结构并调整脚本。

通过这种方式，本项目充当了一个“UI 结构可视化 + 坐标/属性查看器”，而 AutoJs6 则负责实际的自动化执行。

## 6. 常见注意事项（简要）

- **权限问题**：
  - 需要开启无障碍服务；部分功能需要更高权限（以官方文档为准）。
- **分辨率 / 布局差异**：
  - 不同机型或系统版本的界面布局可能不同，建议优先使用基于文本、ID 的选择器，而不是绝对坐标。
- **稳定性与防检测**：
  - 高频操作、过快点击可能导致应用崩溃或触发风控，需要适当增加延时、合理控制频率。
- **法律与合规**：
  - 请仅在符合应用条款和当地法律的前提下使用自动化脚本，对脚本行为自行负责。
